## Notation

Évalue selon ces 3 dimensions :

**Complétude /20** : Le LLM a-t-il couvert toutes les fonctionnalités demandées dans le brief ? (liste, détail, ajout, suppression, recherche, portions dynamiques, seed data)

**Qualité code /20** : Le code est-il lisible, structuré, sans antipatterns évidents ? L'architecture est-elle cohérente avec la stack demandée (FastAPI, SQLite, JS vanilla) ?

**Démarche /20** : Le JOURNAL.md montre-t-il une progression logique ? Le LLM a-t-il anticipé les problèmes, corrigé ses erreurs, fait une auto-évaluation honnête ?

Si JOURNAL.md est absent, la dimension Démarche est notée 0.
Si le validator a échoué (exit 1), pénaliser Complétude proportionnellement aux tests ratés.
